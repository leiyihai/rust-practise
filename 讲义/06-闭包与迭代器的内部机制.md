# 闭包与迭代器的内部机制

## 闭包不是函数指针

每个闭包都是**匿名的、编译器生成的结构体**。它捕获的变量成为结构体的字段：

```rust
let offset = 10;
let add_offset = |x| x + offset;  // 编译器生成一个包含 &offset 字段的匿名结构体
```

闭包自动实现三种 trait 中的一种或多种：

| Trait | 捕获方式 | 调用次数 |
|-------|---------|:---:|
| `FnOnce` | 可能消费（转移所有权） | 一次 |
| `FnMut` | 可变借用 | 多次 |
| `Fn` | 不可变借用 | 多次 |

编译器选择最宽松的那个——这让闭包在大多数场景下"能用就行"，无需手动标注。

## move 关键字：强制转移所有权

```rust
let s = String::from("hello");
let consume = move || println!("{}", s);  // s 被移入闭包
// println!("{}", s);  // 编译错误
```

`move` 常用于跨线程传递闭包（`thread::spawn` 要求 `FnOnce + Send + 'static`）。

## 迭代器的零成本抽象

Rust 的迭代器是**编译时多态**——不是运行时虚调用。当你写：

```rust
let sum: i32 = vec.iter().filter(|x| **x > 0).map(|x| x * 2).sum();
```

编译器会生成等价于手写 for 循环的机器码。没有中间分配，没有虚函数调用。这就是"零成本抽象"的含义。

## IntoIterator：三种迭代方式

```rust
for x in &vec   // 不可变引用迭代，调用 .iter()，类型是 &T
for x in &mut vec  // 可变引用迭代，调用 .iter_mut()，类型是 &mut T
for x in vec    // 所有权迭代，调用 .into_iter()，类型是 T
```

理解这个区别很重要——选错方式可能导致你不小心移动了数据。

## Iterator trait 只要实现一个方法

```rust
trait Iterator {
    type Item;
    fn next(&mut self) -> Option<Self::Item>;
    // 其他 70+ 个方法都有默认实现
}
```

只需实现 `next`，自动获得 `map`、`filter`、`fold`、`collect` 等全部方法。

## 触类旁通：惰性求值

Rust 的迭代器适配器是**惰性**的——你不消费，它不计算。这和很多语言不同：

- **Python** 的 `map`/`filter` 返回迭代器（也是惰性的，Python 3 改了）
- **JavaScript** 的 `Array.map` 返回新数组（立即求值）
- **Java Stream API** 类似 Rust：惰性适配 + 终端操作
- **C# LINQ** 也是惰性的

惰性求值的好处：你可以写 `0..1_000_000_000` 表示一个范围，它不会真的分配 10 亿个元素。
