# 模式匹配的十八般兵器

## match：Rust 的瑞士军刀

```rust
match value {
    Pattern1 => expr1,
    Pattern2 if guard => expr2,  // 匹配守卫
    Pattern3 | Pattern4 => expr3,  // 或模式
    _ => default_expr,  // 通配符
}
```

match 是**穷尽的**——编译器会检查你是否覆盖了所有可能。少写一个分支？编译不过。这是 Rust 防止逻辑遗漏的关键机制。

## 可匹配的模式远比你想象的多

```rust
// 解构结构体
match point {
    Point { x: 0, y } => ...   // x 为 0 时匹配，绑定 y
    Point { x, y: 0 } => ...   // y 为 0 时匹配，绑定 x
    Point { x, y } => ...      // 匹配所有，绑定两个字段
}

// 解构枚举
match skill {
    Skill::FireBall(damage) if damage > 50 => ...  // 高伤害火球
    Skill::FireBall(damage) => ...                 // 普通火球
    Skill::Heal(amount) => ...                     // 治疗
    _ => ...                                       // 其他
}

// 解构切片
match numbers.as_slice() {
    [] => ...              // 空
    [first] => ...         // 单元素
    [first, second] => ... // 两元素
    [first, .., last] => ... // 至少两元素
}

// 范围匹配
match age {
    0..=12 => "儿童",
    13..=17 => "青少年",
    18..=59 => "成年人",
    60.. => "老年人",
}
```

## if let：只要一个分支时的语法糖

```rust
// 等价写法
match opt {
    Some(x) => println!("{}", x),
    _ => (),
}

if let Some(x) = opt {
    println!("{}", x);
}
```

`if let` 还可以带 else，以及配合 `||` 或 `&&`：

```rust
if let Some(x) = a && let Some(y) = b {
    // a 和 b 都是 Some
}
```

## while let：持续解构

```rust
while let Some(top) = stack.pop() {
    println!("{}", top);
}
```

等价于循环 + match，但更紧凑。

## let 语句本身就是模式匹配

```rust
let (x, y, z) = (1, 2, 3);     // 解构元组
let Point { x, y } = origin;   // 解构结构体
```

这能工作是因为 `let` 要求模式是**无可辩驳的**（irrefutable）——必须能匹配所有情况。而 `if let` 接受**可辩驳的**模式。

## 触类旁通：其他语言的模式匹配

- **Haskell/Elm/OCaml**：模式匹配是核心语言特性，比 Rust 更极致
- **Python 3.10+**：有 `match-case`，但弱很多——不检查穷尽性，无借用检查
- **JavaScript**：提案阶段，TC39 还在讨论
- **Go**：没有模式匹配，全靠 `if-else` 和 `switch`

Rust 的模式匹配独特之处在于和所有权系统深度整合——解构时你可以选择移动、引用或忽略字段。
