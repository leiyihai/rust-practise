# 所有权模型深度解析

## 三条规则，一个世界

1. Rust 中每一个值有且只有一个**所有者**
2. 值在所有者离开作用域时被 drop
3. 同一时刻，要么有一个可变引用，要么有任意多个不可变引用——不能兼得

这三条规则编译期强制执行，无需 GC，无需手动 free。

## 移动语义：赋值即转移

```rust
let s1 = String::from("hello");
let s2 = s1;   // s1 被"移动"了，不再可用
// println!("{}", s1);  // 编译错误！
```

这是 Rust 和几乎所有其他语言的本质区别。在 C++ 中这是拷贝（或移动后源对象仍可用），在 Python 中这是增加引用计数。Rust 选择默认移动，出于两个原因：
- 避免双重释放（两个变量都认为自己拥有堆内存）
- 零开销（不需要引用计数）

## Copy 类型：赋值时自动克隆

实现了 `Copy` trait 的类型在赋值时会**按位复制**而非移动。基本类型都是 Copy：`i32`, `f64`, `bool`, `char`, `&T`（引用本身）, `[T; N]`（当 T 也是 Copy）。

判断一个类型是不是 Copy：它在栈上的位模式可以直接复制，不涉及堆资源。

## 借用：让别人用，但别拿走

```rust
fn len(s: &String) -> usize { s.len() }  // 不可变借用
fn push(s: &mut String, c: char) { s.push(c) }  // 可变借用
```

- `&T` —— 共享引用，可以复制（实现了 Copy），可以同时存在多个
- `&mut T` —— 独占引用，不可复制，同一时间只能存在一个

## 生命周期省略规则

大多数情况下你不需要标注生命周期。编译器有三条省略规则：

1. 每个引用参数获得自己的生命周期
2. 如果只有一个输入生命周期，它被赋给所有输出生命周期
3. 如果有 `&self` 或 `&mut self`，它的生命周期被赋给所有输出生命周期

只有当这三条不够用时，编译器才会让你显式标注。

## 触类旁通：为什么其他语言不需要这些

- **Java/C#/Go**：有 GC，运行时追踪引用，程序员不用操心
- **C**：没有规则，全靠程序员自律——忘记 free 就是内存泄漏，继续用已释放的指针就是 UB
- **C++**：有 `unique_ptr`、`shared_ptr` 模拟所有权，但编译器不强制
- **Rust**：编译期强制执行，零运行时开销——写的时候痛苦，跑的时候放心

这种"编译时多花时间，运行时安全高效"的取舍，是 Rust 最核心的设计哲学。
