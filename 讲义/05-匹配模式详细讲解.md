## 1. match 核心定位与基础语法
match 是 Rust 完备的分支分发工具，被称作 Rust 的「瑞士军刀」，区别于 C/C++ 的 `switch-case`，支持任意类型、任意模式匹配，无分支穿透。

### 基础语法模板
```rust
match 待匹配值 {
    模式1 => 表达式1,
    模式2 if 匹配守卫条件 => 表达式2,
    模式3 | 模式4 => 表达式3,
    _ => 默认兜底表达式,
}
```
- `=>` 后为执行体，多行代码用 `{}` 包裹，末尾不需要分号；
- 匹配规则：自上而下依次匹配，命中第一个符合分支后立即终止。

## 2. match 核心特性：穷尽性检查（Rust 安全关键设计）
match 编译器会静态校验：**枚举所有变体必须被全部覆盖**，遗漏分支直接编译报错，杜绝逻辑遗漏、空指针、未处理错误等隐患。

### 示例：Option 穷尽校验
```rust
let opt: Option<i32> = Some(10);
match opt {
    Some(v) => println!("取值：{}", v),
    None => println!("空值"),
}
```
删除 `None` 分支，代码直接编译失败。

### 通配符 `_`
不想逐一枚举所有场景，使用 `_` 捕获剩余所有情况，充当万能兜底分支，必须放在所有分支末尾：
```rust
let n = 7;
match n {
    1 => println!("数字一"),
    2 => println!("数字二"),
    _ => println!("其他数字"),
}
```

## 3. match 进阶语法
### 3.1 多模式或匹配 `|`
一个分支同时匹配多个并列模式，竖线分隔：
```rust
let ch = 'a';
match ch {
    'a' | 'b' | 'c' => println!("属于abc小写字母"),
    '0'..='9' => println!("数字字符"),
    _ => println!("其他字符"),
}
```

### 3.2 匹配守卫 `if 条件`
模式匹配成功后，额外追加布尔条件二次过滤，格式：`模式 if 条件 =>`
```rust
let score = Some(60);
match score {
    Some(n) if n >= 60 => println!("成绩及格"),
    Some(n) => println!("不及格，分数：{}", n),
    None => println!("无成绩数据"),
}
```

### 3.3 范围模式
`..=` 闭区间（包含首尾）、`..` 左闭右开区间，批量匹配连续数值：
```rust
let age = 25;
match age {
    0..=12 => println!("儿童"),
    13..=17 => println!("青少年"),
    18..=60 => println!("成年人"),
    61.. => println!("老年人"),
}
```

## 4. 复合类型解构匹配
match 支持一边匹配，一边直接解构绑定内部字段，无需额外赋值。

### 4.1 解构结构体
```rust
#[derive(Debug)]
struct Point {
    x: i32,
    y: i32,
}

let p = Point { x: 0, y: 10 };
match p {
    // 限定x固定为0，绑定y到变量y_val
    Point { x: 0, y: y_val } => println!("x轴原点，y值：{}", y_val),
    // 限定y固定为0，绑定x到变量x_val
    Point { x: x_val, y: 0 } => println!("y轴原点，x值：{}", x_val),
    // 无限制，完整绑定两个字段
    Point { x, y } => println!("普通坐标：x={}, y={}", x, y),
}
```

### 4.2 解构自定义枚举（游戏技能场景）
```rust
enum Skill {
    FireBall(u32), // 火球术：附带伤害数值
    Heal(u32),     // 治疗术：附带回复血量
    Dash,          // 闪现：无附加参数
}

let sk = Skill::FireBall(80);
match sk {
    Skill::FireBall(dmg) if dmg > 50 => println!("高额火球，伤害{}", dmg),
    Skill::FireBall(dmg) => println!("普通火球，伤害{}", dmg),
    Skill::Heal(val) => println!("治疗血量：{}", val),
    Skill::Dash => println!("执行闪现位移"),
}
```

### 4.3 解构切片/数组
匹配空数组、单元素、首尾元素等场景：
```rust
let arr = [10, 20, 30, 40];
match arr.as_slice() {
    [] => println!("空数组"),
    [a] => println!("仅单个元素：{}", a),
    [first, .., last] => println!("首元素{}，尾元素{}", first, last),
    _ => (),
}
```

## 5. match 是表达式，支持返回值赋值
Rust 中 match 整体属于表达式，每个分支返回同类型数据，可直接赋值给变量，C/C++ switch 不具备该特性。
```rust
let score = 78;
let level = match score {
    0..=59 => "不及格",
    60..=79 => "及格",
    80..=100 => "优秀",
    _ => "非法分数",
};
println!("评级结果：{}", level);
```

## 6. 语法糖：if let / while let
是 match 的简化写法，牺牲穷尽检查，简化单边分支代码。

### 6.1 if let：只关心单个变体
只处理目标匹配分支，剩余场景直接忽略，不用写 `_ => ()` 空分支。
等价对照：
```rust
let opt = Some(99);
// 完整版match
match opt {
    Some(v) => println!("数值：{}", v),
    _ => (),
}

// if let 简写
if let Some(v) = opt {
    println!("数值：{}", v);
}
```
支持 `else` 分支，也可多条件联用：
```rust
if let Some(x) = a && let Some(y) = b {
    // a、b 同时为 Some 才执行
}
```

### 6.2 while let：循环持续解构
循环匹配，模式无法命中时自动终止循环，常用于连续弹出栈、迭代取值：
```rust
let mut stack = vec![1, 2, 3];
// 不断弹出元素，栈空返回None，循环结束
while let Some(val) = stack.pop() {
    println!("弹出元素：{}", val);
}
```

## 7. let 语句本质也是模式匹配
常规变量绑定、解构赋值，底层都是模式匹配，这类模式**一定匹配成功**。
```rust
// 元组解构
let (a, b) = (10, 20);

// 结构体解构
let Point { x, y } = Point { x: 5, y: 6 };
```

## 8. 可辩驳模式 vs 无可辩驳模式
| 分类         | 说明                                   | 使用场景                     |
|--------------|----------------------------------------|------------------------------|
| 无可辩驳模式 | 编译期确定一定匹配成功，不会失败       | `let` 绑定、函数参数解构     |
| 可辩驳模式   | 存在匹配成功、匹配失败两种可能性        | `match`、`if let`、`while let` |

## 9. 实战选型建议（适配你的背包/银行/技能练习）
1. **必须完整处理所有分支、错误不能遗漏**（查询账户、存取款）
   - Option / Result 枚举 → 标准 `match`，编译器强制穷尽检查
2. **仅关心成功分支，失败直接忽略**
   - `if let Some(xxx) = ...` / `if let Ok(val) = ...`
3. **循环持续取值，直到数据耗尽**
   - 栈遍历、逐个取出容器元素 → `while let`

示例参考：
```rust
// 账户查询（穷尽match）
match bank.query_acc(1) {
    Some(acc) => {}
    None => {}
}

// 存款操作
match bank.deposit(1, 1000.0) {
    Ok(new_balance) => {}
    Err(err_msg) => {}
}
```

## 10. 模式匹配与所有权系统结合
解构时可选择引用/移动所有权，不会强行转移原变量所有权：
```rust
let opt = Some(String::from("测试字符串"));
// 引用解构，不拿走所有权，opt后续仍可正常使用
match &opt {
    Some(s) => println!("{}", s),
    None => (),
}
```
这也是遍历 `iter()`（不可变引用）、`iter_mut()`（可变引用）时，能直接在 match 中解构引用的底层原理。
