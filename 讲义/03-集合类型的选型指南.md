# 集合类型的选型指南

## 数组 [T; N] vs 切片 &[T] vs 向量 Vec<T>

```text
数组 [i32; 5]     —— 栈上，编译期确定大小，不可变长度
切片 &[i32]        —— 胖指针（ptr + len），某段连续数据的视图
向量 Vec<i32>      —— 堆上，运行时可变长度，拥有数据
```

关键认知：**切片不拥有数据**。`Vec` 可以自动解引用为切片，所以所有接受 `&[T]` 的函数也能接受 `&Vec<T>`。

## Vec 的性能特点

- 连续内存，缓存友好，随机访问 O(1)
- 尾部 push 均摊 O(1)，超出容量时 2× 扩容 + 复制
- 中间插入/删除 O(n)
- 如果你知道大概的容量，用 `Vec::with_capacity(n)` 避免多次扩容

## HashMap —— 当顺序不重要时

```rust
use std::collections::HashMap;
let mut map = HashMap::new();
map.insert("火球术", 30);
let dmg = map.get("火球术");  // Option<&i32>
```

- 平均 O(1) 查找，但常数比 Vec 大
- 不保证顺序（迭代顺序可能每次运行不同）
- 需要 key 实现 `Hash + Eq` trait
- 用 `entry()` API 优雅处理"存在就更新，不存在就插入"

## 触类旁通：其他集合速查

| 集合 | 特点 | 典型场景 |
|------|------|---------|
| `VecDeque<T>` | 双端队列，头尾操作 O(1) | 队列/栈 |
| `LinkedList<T>` | 双向链表 | 几乎不用（缓存不友好） |
| `BTreeMap<K,V>` | 有序映射，O(log n) | 需要顺序遍历 |
| `HashSet<T>` | 就是 `HashMap<T,()>` | 去重、成员检查 |
| `BinaryHeap<T>` | 最大堆 | 优先级队列 |

## 迭代器的消费与适配

迭代器方法分两类：

- **适配器** (lazy)：`map`、`filter`、`take`、`skip`、`rev`、`enumerate`、`chain`、`zip` —— 返回另一个迭代器，不做实际计算
- **消费器** (eager)：`collect`、`for_each`、`count`、`sum`、`fold`、`find`、`any`、`all` —— 真正执行遍历

这意味着你可以链式组合适配器，直到最后才触发计算——零中间分配。

```rust
let result: Vec<_> = (0..100)
    .filter(|x| x % 2 == 0)   // 还没计算
    .map(|x| x * x)            // 还没计算
    .take(5)                   // 还没计算
    .collect();                // 这里才开始执行
```
